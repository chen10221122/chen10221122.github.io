接上一节。我们打印出来event.__proto__.constructor，constructor是个Class。

回顾之前vue的例子，他的event是MouseEvent，是原生的。

所以说，我们有必要怀疑，React组件里面的事件，是否真的挂载到了当前的元素上。

我们有一个方法可以获取原生的：

此处的event.nativeEvent就是原生事件。

需要注意他的currentTarget是绑定事件的元素，是 #document，这一点就和Vue完全不一样了。

Vue：触发的节点和绑定的节点是一样的。

React：触发的节点是原来的，绑定的节点都是#document

总结：

  1. event 是 SyntheticEvent ，模拟出来 DOM 事件所有能力
  2. event.nativeEvent 是原生事件对象
  3. 所有的事件，都被挂载到 document 上
  4. 和 DOM 事件不一样，和 Vue 事件也不一样

7-11 React组件生命周期

重要

单组件生命周期

React 组件生命周期图示： http://projects.wojtekmaj.pl/react-lifecycle-methods-diagram/

也可以对比一下vue的生命周期： https://cn.vuejs.org/images/lifecycle.png

  • 挂载时：
    • constructor：初始化，类似于vue的beforeCreate
    • render：渲染界面的时候。render在Vue中是没有的，因为Vue的模板时直接写在template里面的，没有函数去返回模板，
    • componentDidMount：和vue的mounted是对应的。vue里面的beforeMounted、created这些，React里面是没有的，也不是那么常用。之前说，Vue的ajax请求一般是在mounted里面，对应React里面，应该放在 componentDidMount里。
  • 更新时：
    • componentDidUpdate：对应Vue的updated。也是要触发setState()，再触发render，会触发componentDidUpdate
    • shouldComponentUpdate (不常用)：可以控制它更新的时候是否继续渲染，有点时候是没有必要继续渲染的
  • 销毁时。
    • componentWillUnmount：对应Vue的beforeDestroy。可以在里面做一些如销毁自定义事件、销毁DOM事件、定时器等。

其他的生命周期不常用，应用场景不多。最好可以把图画出来，两者可以结合起来看。
