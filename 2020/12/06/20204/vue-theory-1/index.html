3-6、生命周期（单个组件）

挂载、更新、销毁

  • vue官网： https://cn.vuejs.org/v2/guide/instance.html#%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F%E5%9B%BE%E7%A4%BA
  • create：页面还没有开始渲染，但是vue的实例已经初始化完了
  • mounted：页面已经渲染完成

  • beforeDestroy：解除绑定、销毁子组件及事件监听器

  • 生命周期（父子组件）

刚才的示例：父组件index组件，子组件：input组件、list组件。给每个生命周期console.log()，结果如下：

  • index created
  • list created
  • list mounted
  • index mounted

创建(初始化）是从外到内的，但是渲染是从内到外的，因为父组件可能调用子组件的东西。所以子组件先渲染。

点击添加一条数据，update 、before update 打印结果：

  • index before updata
  • list before update
  • list updated
  • index updated

点击add的时候，触发父组件的事件，父组件的data首先被修改，触发更新；

更新list，只有list更细、渲染完了之后，index才会渲染完成

3-14、vue如何抽离公共逻辑（mixin）

mixin

  • 多个组件有相同逻辑，抽离出来。
  • mixin并不是完美的解决方案，会有一些问题
  • Vue 3 提出的Composition API旨在解决这些问题

代码演示：

MixinDemo.vue:引入了一个mixinjs，

mixin.js:导出了一个对象。

mixins，可以把引入的js的内容，通过配置的方式拿过来，和当前信息混合。

多个组件如果有公用的逻辑，可以不同的组件，都引入公共的部分复用，这就是他的价值。

我们的设计原则是引用，而不是大量重复的代码。

mixin的问题

  • 变量来源不明确，不利于阅读
  • 多个mixin可能会造成命名冲突（冲突它也不报错，直接就覆盖了）
  • mixin和组件可能出现多对多的关系，复杂度较高

3-15、vue高级特性知识点小结

总结

  • 自定义v-model

像类似于颜色选择器的，这种东西，我们可以通过，自定义v-model的方式，来把一个data，进行双向数据绑定；

具体的使用，包括event事件，props,还有model的配置等，

  • $nextTick

这个的关键在于，vue是异步渲染的，我们要通过$nextTick，去触发异步渲染完之后、DOM元素变化之后的回调。

比如想在data改变之后，立刻拿到元素，就在$nextTick里面去拿。

  • slot

插槽，包括

基本的插槽使用：父组件把一些标签内容，传递给子组件，子组件在找个地方插进去

作用域插槽：子组件的一个data，能传给父组件来操作，

具名插槽：多个插槽的时候，如何一对一，对应起来

  • 动态、异步组件

动态组件：在一些像新闻详情页面、tab页面，通过:is动态属性，传入组件名称，加载不同组件

异步组件：在加载大的组件的时候，我们用import函数，去异步加载，而不是同步加载

  • keep-alive

就是我们要缓存一部分组件，比如tab页面切换、路由

  • mixin

多个组件有相同逻辑的时候，我们可以用mixin抽离，他也有很多问题，比如变量来源不明确、两者之间可能重复、可能出现多对多，关系复杂

综上：

  • 可以不太深入、但必须知道
  • 熟悉基本用法，了解使用场景
  • 和自己项目经验结合
