5.1、v-for为何使用key

v-show和v-if的区别

  • v-show 通过 CSS display 控制显示和隐藏
  • v-if组件真正的的渲染和销毁，而不是显示和隐藏
  • 频繁切换显示状态用v-show，否则用v-if

为何在v-for中使用key

  • 必须用key，且不能是index和random
  • diff算法中通过tag和key来判断，是否是sameNode
  • 减少渲染次数，提升渲染性能

描述组件渲染和更新的过程

如上图：vue原理的三大模块，第一个就是响应式，就是监听属性变化，就是紫色的部分。第二个就是模板渲染，就是黄色的部分， 虚拟DOM就是左下角这块。

执行Render函数的时候会触发 “touch”，触发getter，然后收集依赖，到watcher里面；我们触发更改的时候会触发setter，setter会通知（Notify）watcher，看看是不是之前已经收集过了，若收集过了就触发re-render，重新渲染。

5.2、组件data为何是函数

双向数据绑定v-model的实现原理

模板编译实现的时候，已经总结过了：

  • input 元素的value=this.name
  • 绑定input事件： this.name=$event.target.value
  • data更新触发re-render

绑定v-model后，input元素会添加一个事件 ，每次input的时候，都会把当前value的值，赋值给this.name。

需要了解 v-model在模板编译之后产生的是什么

对MVVM的理解

最好能把这个图画出来。关键在于 view、model、viewModel，ViewModel里面还有DOM Listeners 和 Directives

computed有何特点

  • 缓存，data不变不会重新计算
  • 合理使用，提高性能

为何组件data必须是一个函数？

这个问题其实是js代码的一个问题， 因为看似export default {}是一个对象，实际上vue文件在编译出来之后，vue是一个class，是一个类。

组件是一个class，我们在其他地方使用的时候，对这个组件实例化。我们在实例化的时候，执行这个data。如果这个data不是一个函数，那每一个组件的实例，数据都一样了，数据也就都共享了。

这样多个地方引用这个组件，这些data就在闭包之中，data就互不影响。

ajax请求应该放在哪个生命周期

  • mounted
  • JS是单线程的，ajax一步获取数据
  • 放在mounted之前没有用，只会让逻辑更加混乱
