3-8、生命周期

一个组件从出生到死亡的全过程。生命周期会暴露出一系列的钩子函数，供开发者在对应阶段操作

vue2生命周期： https://cn.vuejs.org/v2/guide/instance.html#%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F%E5%9B%BE%E7%A4%BA

  • 新建Vue实例
  • 初始化事件&生命周期
  • beforeCreate
  • 初始化注入&校验
  • created
  • 一系列render的过程
  • beforeMount :挂载DOM之前
  • mounted:实例挂载DOM之后
  • beforeUpdate:挂载完毕，当data被修改
  • updated：虚拟DOM从新渲染并应用更新
  • beforeDestroy：当调用vm.$destroy()函数时，（实例被卸载）
  • desrtoyed:解除绑定、销毁子组件及事件监听器

vue3中大体保持不变，beforeDestroy改成beforeUnmount，desrtoyed改成unmounted

这样写可以更好的语义化，对应mounted，更形象

vue3生命周期： https://v3.vuejs.org/guide/instance.html#lifecycle-diagram

  • beforeCreate -> 使用 setup()
  • created -> 使用 setup()
  • beforeMount -> onBeforeMount
  • mounted -> onMounted
  • beforeUpdate -> onBeforeUpdate
  • updated -> onUpdated
  • beforeDestroy -> onBeforeUnmount
  • destroyed -> onUnmounted
  • activated -> onActivated
  • deactivated -> onDeactivated
  • errorCaptured -> onErrorCaptured
  • onRenderTracked (新增)
  • onRenderTriggered （新增）

beforeCreate 、 created和setup，几乎同时进行。新增两个调试用的debugger钩子，观察数据变化。

3-13、Vue2遇到的难题

  • Vue2对于 typescript 的支持非常的有限

上面我们解释了Vue3如何重新定义逻辑的重用和抽取的，这也是compositionAPi的最重要的任务，除此之外，Vue3还加强了对typescript的支持。

Vue2依赖this上下文对象往外暴露属性，但是在组件中，this比普通的 javascript Object多了很多“黑魔法”，换言之，Vue2当初就没有考虑到集成typescript，我们看下Vue3是如何解决这个问题的。

Vue2和typescript结合一般会使用vueClass的写法，或者 vue.extends来继承typescript；Vue3提供了一个新方法component，称之为 defineComponent，他并没有实现任何逻辑，只是把传入的Object返回，他的存在是让传入的整个对象获得类型，换言之，他是完全为了服务typescript而存在的。

如果我们导出普通对象，自然就无法获取提示，但若使用 defineComponent 进行包裹，代码过程中则可以提示，包括Vue2原属性和Vue3新增属性

setup可以接受两个参数，props 和 context，

  • props可以访问组件传入的props属性，而且会自动推论；同时他是响应式对象，可以watch他的值。
  • setup中无法访问Vue2中重用的this对象，context提供了三个最常用的属性 attrs slots emit，这几个值会自动同步
