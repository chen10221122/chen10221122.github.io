10-6 webpack如何抽离公共代码和第三方代码

打包的时候，很多公共的部分，需要抽离出来。我们要避免重复，避免重复引用。

比如修改了业务代码，引入的第三方模块就会重新打包，hash值也会边，但是第三方模块的内容本身是没变的，这样浪费性能

commom.js和dev.js里面无需改动，不需要压缩、拆分、合并、分析等，这样可以构建的更快，提高开发效率。

prod.js里面的改动：

1.增加splitChunks配置。chunk是一段代码。

  • 分组分了两组，第一组叫第三方模块，名字叫vender，第二组叫公共的模块，名字叫common;

  • priority：权重。比如第三方模块和公共模块，出现了冲突，通过这个来区分。

  • test：用来看是否命中。一般第三方包都在/node_modules/里，比如lodash，他就会被打包到vendor里

  • minSize：有些包体积很小，无需单独拆开，可以通过限制minSize，太小了就不单独打包，可按需使用，如3kb

  • minChunks:1 针对第三方模块，只要被引入了一次，就单独打包。公共的模块，可以配置大一点，被引用多次在单独拆出来。

  1. npm run build试一下，查看dist下的index.html

  1. other.html里面并没有引用第三方资源，但是打包后也引用了 vendor.js

为什么这样呢？我们看下common.js里面的配置：

  1. plugins里面我们配置了chunks: ['other', 'vendor'，'common']，而vender和common就是我们上面配置的name,表示每个入口下需要哪些chunk

如果我们other里面不配置vender，打包后的dist目录里other.js，就不会引入vendor.js了。

chunk是什么

chunk是拆分的代码块的意思。生成chunk的地方包括entry和splitChunks。entry产出的chunk可以在plugins: []里面使用

10-8 module chunk bundle 的区别

这三个东西是比较基本的概念，

  • moudule - 各个源码文件，webpack中一切皆模块
    • 对应源码中，src文件夹下，除index.html外，都是模块，包括图片、css、js等可以引用的东西。
  • chunk - 多模块合并成的，如 entry import() splitChunk 等可以定义chunk
    • 根据定义的文件名，可以分析依赖，然后生成一坨代码的集合
    • 比如entry里面，根据index.js生成index这个chunk，这个chunk里面不一定就是index.js这个文件，还可能会包括index.js引用的其他文件
    • 比如 import()产出的chunk，也包含其他文件
    • 比如 splitChunk 中定义的chunk，就是多个文件合成的代码集合
  • bundel - 最终的输出文件
    • chunk可以理解为在内存中的文件，还没有输出，bundel是每个chunk对应的输出文件。
  • 概念问题最好结合使用回答，比如生成chunk的方式。
