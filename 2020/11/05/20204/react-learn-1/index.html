二、页面加载过程

一个html页面是由很多资源组成的，如最基本的HTML本身是一个资源，js、css、图片、文本、json等资源。这些资源都是通过http或https请求从服务器拿过来的。

1、资源加载过程

在浏览器输入了地址之后，究竟都发生了什么？

请求资源会有一个url,第一步是解析url,

第二步是拿到解析出来的域名，去dns服务器上查找该域名对应的ip,

第三部是带着所有的请求信息去这个ip地址上请求资源，然后再从服务器上把返回的资源下载下来

最后一步是浏览器拿到这些资源以后，根据不同的类型，做不同方式的解析。

5、浏览器解析

刚开始是拿到一个html文件，在加载html的同时，开始构建DOM树：遇到一个html节点，就把他放到DOM树里，

假如构建的时候遇到一个js文件，那么构建DOM树的工作就要先停下来，先让js加载或执行。js有这么高的优先级是因为他可以操作DOM元素，这可能造成之前构造的DOM树都白干了，所以遇到js的时候，会让js先执行。

js执行完之后，接着构建剩下的DOM树， 如果遇到style标签，就不会造成阻塞了，DOM树的构建和样式的加载会并行的进行。

之后会构造渲染树,渲染树是DOM树和CSS样式表结合的产物，这个渲染书在不同的浏览器构造的机制也不太一样，比如chorme用的webkit内核，是在原来的DOM树上附属一些样式，而firefox是根据DOM树和样式表，重新构建出一颗新的渲染树出来。渲染树构建出来，每一个布局的大小、样式也都确定了。

接下来就是根据每个元素的大小和布局方式，计算出每个元素的实际位置，这就是布局的过程

最后一步是绘制。绘制过程就是调用浏览器负责显示的部分，将元素按照对应的位置和样式，绘制到屏幕上，这样整个浏览器的解析过程就算完事了

这是一个比较理想的模型。实际中各种浏览器为了优化加载速度，会有更多复杂的处理方法，比如说DOM树的构建和渲染树的构建可能是同是进行的，还有的浏览器有预加载的功能。所以正常加载的时候大都不是按照这个顺序解析的。

1、ES6?

什么是ES6? 从表面来看可以确定，这是一个什么东西的第六个版本，就像谍影重重5、生化危机7是一个道理。

他是ECMA Script的缩写，这个ECMA是一个专注于制定计算机行业规范的世界组织，ECMAScript,就是标准化的脚本程序设计，而ES6就是这个规范的第六个大版本。这个版本的js是在2015年正式发布的，所以ES6也被称为ES2015。

提升JS编写大型应用的能力。比如这次升级加入了模块化的概念，会在一些语法上做了细化或优化，还丰富了一些对象的方法，这些内容虽然在其他版本都可以通过其他方式实现，比如说模块化我们可以借助node.js来做commonjs形式的模块化，面向对象我们可以用function代替。但是现在ECMA把这些内容放入了官方的规范。

四、本地存储

1、cookie 、session

前后端使用http协议，但是http是无状态的，就是得到一个请求，返回一个相应，而不关心请求者的身份，http也不是一种持持久性的链接，而对于请求来说，记住某个用户的身份特征是比较困难的。

cookie的出现是为了在用户端保存请求的机制。浏览器会在操作系统里开辟一个文件，专门存放cookie，这个文件只要不删除就会一直存在。而在每次请求时，都会在http头里面带上这个cookie信息，后端就能知道这个http请求是谁发来的，

而cookie机制又是一个字符串，他的格式是逗号分隔的多个key-value字段

cookie存储在系统的本地文件里，是可以通过浏览器操作的为数不多的文件之一，但出于安全考虑，这个cookie文件，一般是加密的，只有浏览器可以操作他。如果我们直接打开这个文件，就会发现他是一堆加密后的文本，不能直接阅读里面的内容

cookie还有域名和路径的限制，一个请求只能操作自己有权限的cookie。

cookie里面有如下几个字段。

和cookie对应的还有session机制。

session是在服务端记录请求者身份用的，这个机制就是在接到http请求以后，服务端生成一个seddionId,然后把这个sessionId装到用户的cookie里，这样同一个用户再有请求，就可以根据带过来的cookie，知道这个请求时谁发的。

当然sessionId并不一定都是用cookie来实现的，也可以是放在请求参数里，或者是在http头里面，开辟一个token字段。

没有设置过期时间的cookie，只会存在于一次会话中，浏览器一关闭就结束了，而cookie里这个一般不会设置过期时间的sessionId也就消失了
