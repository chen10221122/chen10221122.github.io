HTTP_REFERER的作用
更新于 2021-04-23 http

在修改之前人写的项目时发现，直接点击 a 标签打开新页面会 403，在浏览器地址栏直接输入则不会，故考虑到后台对 HTTP_REFERER 做了判断。

什么是 Referer

HTTP Referer 是 header 的一部分，当浏览器向 web 服务器发送请求的时候，一般会带上 Referer，告诉服务器该网页是从哪个页面链接过来的，服务器因此可以获得一些信息用于处理。

就告诉你从哪里过来的。如下图:

Referer 的作用

1、防外链.

比如我只允许我自己的网站访问我自己的图片服务器，那么图片服务器每次取到 Referer 来判断一下是不是我自己的域名，如果是就继续访问，不是就拦截。

下列情况是从浏览器的地址栏正常取得 Request.ServerVariables(“HTTP_REFERER”)：

  • 直接用<a href="">
  • 用 Submit 或<input type="image">提交的表单(POST 或 GET)
  • 使用 JavaScript 提交的表单(POST 或 GET)

Request.ServerVariables(“HTTP_REFERER”)不能正常取值的情况：

  • 从收藏夹链接
  • 单击主页或自定义的地址
  • 在浏览器中直接输地址
  • 使用 JavaScript 的 Location.href 或者是 Location.replace()
  • <%Response.Redirect%>
  • <%Response.AddHeader%>或者是<mete http-equiv="refresh">转向
