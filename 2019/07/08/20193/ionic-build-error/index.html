ionic cordova build 报错解决 （ionic 4.3.0 Cordova 9.0.0）
更新于 2021-04-23 ionic

昨天更新了 ionic 和 cordova，发现编译的时候又出问题了。问题原因好像是新版本的 cordova 需要使用 gradle。遂记录下来，以便下次遇到时查阅。

下面是当前配置和build成功后的图片

如果忘记了ionic cordova platform add android，就会出现下面这个错误，

1、FAILURE: Build failed with an exception

最开始是提示没有gradle，会提示你登陆他的官网 ；

之后下载并配置了环境变量，

2、Error:Unable to start the daemon process/Unable to start the daemon process. This problem might be caused by incorrec

好像还跟着什么内存错误的提示。解决方法：进入 C:\Users\Administrator.gradle 路径下(其中 Administrator 是自己的用户名)，新建 gradle.properties 文件，并写入

3、然后提示网络超时的问题，连接 vpn 就可以了。

4、”Could not resolve all files for configuration ‘:classpath’.” error

因为中国国内的网络环境会导致 Andrioid Studio 下载的 gradle 版本的某些文件丢失。

解决方法如下:

  • 1、删除用户目录 C:\user\Administrator.gradle\wrapper\dist \ grade -4.4 文件夹中的所有文件
  • 2、到 gradle 官方网站下载 v4.4 gradle zip 包并复制到 Step1 的文件夹
  • 3.Restart Android Studio

相关连接：http://services.gradle.org/distributions/
https://developer.android.google.cn/studio/releases/gradle-plugin#updating-plugin
