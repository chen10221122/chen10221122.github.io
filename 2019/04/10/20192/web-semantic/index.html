web语义化

一、什么是Web语义化

Web语义化是指用合理HTML标记以及其特有的属性去格式化文档内容。通俗地讲,就是对数据和信息进行处理,使得机器可以理解。

就是不要全都用div+span撸完整个网页，那么多标签偏偏都用这两个干嘛；

二、Web语义化的由来

内容的语义表达能力和 AI 的智能程度决定了机器分析处理 Web 内容能力的高低，所以语义化的发展其实就是语义规范的发展以及人工智能的发展结合的产物。


`

三、Web语义化的优势和劣势

1、清晰的页面结构，加载更快。
2、支持更多的设备。
3、有利于SEO。
4、便于团队开发和维护。

四、Web语义化应用

1、布局

<header> 页眉，一组介绍性描述或导航信息，目录 / 搜索框 / logo 。
<h1-h6> 标题，提纲信息。
<nav> 导航、菜单，可以帮助读屏器迅速获得导航内容，可以省去很多渲染直接跳到导航位置。
<article>独立的文档、页面、应用、站点。
可以单独发布、重用，一般应用于帖子、评论、文章等
<section> 用于区分相同主题的不同内容，屏幕阅读器以此梳理出提纲。
<address> 代表与最近父级<article>或整个文档关联的联系人信息，比如作者信息、内容主人公原型等。
<aside> 表示与周围内容关系不太密切的内容，通常表现为侧边栏内容或引述内容，比如广告，或是嵌在文章内部的名人名言，悬浮的聊天窗口等。
<footer> 页脚，一般用来描述作者信息、相关文档、版权信息。


2、内容

<main> 文档的主内容 / 应用的核心功能，全页面唯一。
<div> 无语义，任何情况都适用，也是页面容器优先级最低的选择。
<figure> 比较独立的、被主要内容引用的部分。如插图 / 图表 / 照片 / 代码 ,通常搭配<figcaption>做标题
<ol> <ul> 有序 / 无序列表，以行的方式展现内容，选择的依据是 <li> 的出现顺序是否影响语义的表达

<li> 元素的 value 属性代表该列表项的序号值
<details> 标签用于描述文档或文档某个部分的细节，其标题用<summary> 标签。
<blockquote> 引用的来自其他来源的内容，cite属性代表引用url，署名要放在外面，一般配合<cite>标签。
<pre> 表示已排版的内容，空格、换行都会完整呈现。
<hr> 不同主题内容间的分隔符，比如故事场景的转换。
<p> 段落，用来展现内容的最低优先级选择。
<dl> 名值对的集合。<dt>是名，<dd>是值。一般用于术语定义表、元数据定义表、FAQ、商品信息等等。

<form> 表单的容器。
<input> 除了div，最被人熟知的标签。可用类型达20种之多。
<label> 标签为 input 元素定义标签（label）。label 元素不会呈现特殊的样式，但如果点击 label 元素内的文本，会切换到控件本身。<label> 标签的 for 属性等于相关元素的 id 元素。
<datalist>标签定义选项列表。使用 input 元素的 list 属性来绑定 datalist，定义 input 可能的值。简易版comboBox。

<form> 表单的容器。
<input> 除了div，最被人熟知的标签。可用类型达20种之多。
<label> 标签为 input 元素定义标签（label）。label 元素不会呈现特殊的样式，但如果点击 label 元素内的文本，会切换到控件本身。<label> 标签的 for 属性等于相关元素的 id 元素。
<datalist> 标签定义选项列表。使用 input 元素的 list 属性来绑定 datalist，定义 input 可能的值。简易版comboBox。

<table> 标签定义 HTML 表格。
一个简单的 HTML 表格包括 table 元素，一个或多个 tr、th 以及 td 元素。
更复杂的 HTML 表格也可能包含 colgroup, col, caption, thead, tfoot, tbody 等元素。
<tr> 标签定义表格中的行。
<th>、<td> 标签定义 HTML 表格中的单元格。
HTML 表格有两种单元格类型：
<th>标签定义 HTML 表格中的表头单元格（包含头部信息），其文本呈现为粗体并且居中。
<td>标签定义 HTML 表格中的标准单元格（包含数据），其文本是普通的左对齐文本。
<colgroup> 标签用于对表格中的列进行组合，以便对其进行格式化。
<col> 标签为表格中的一个或多个列定义属性值。
<caption> 标签定义表格的标题。
每个表格只能定义一个标题，通常标题居中显示在表格上方。
thead、tfoot 以及 tbody 元素对表格中的行进行分组。（内部必须拥有 <tr> 标签）
<thead> 定义表格的表头。
<tbody> 定义表格的主体。
<tfooter> 定义表格的页脚。
如果使用 thead、tfoot、tbody 元素，必须使用全部的元素。且必须在 table 元素内部使用这些标签。

3、文本

<em> 表示侧重点的强调，强调级别由 em 的嵌套个数决定。em 的位置不同，文本本身含义不同(能穿多少穿多少)，屏幕阅读器解析为斜体。
<stong> 表示内容的重要性，重要程度由 strong 的嵌套个数决定。strong 的位置不同，文本本身含义不变(把我妈和媳妇救上来)，屏幕阅读器解析为粗体。
<i> 表示另一种叙述方式。画外音 / 分类学名词 / 外来语片段 / 舞台指示 / 船名。(室友交了女朋友，我们全寝室都喜(nei)大(xin)普(beng)奔(kui))
<b> 不再只是「粗体」，表示某种需要引起注意却又没有其他额外语义的内容，摘要中的关键词 / 评介中的产品名称 / 文章的开篇内容。
<small> 不再只是「小字」，用来描述必须作出说明但又不想占过多篇幅的内容(请以实物为准，图片仅供参考)，免责声明 / 许可证声明 / 注意事项。
<s> 不再只是「带删除线的文字」，表示不再准确或不再相关的内容(打折50，原价100)，与 del 元素含义不同。
<u> 不再只是「带下划线的文字」，表示中文专名 / 拼写检查的错误内容 如人名、地名、朝代名、国名、机构名等。
<del> 标签定义文档中已删除的文本。
<ins> 标签定义文档的其余部分之外的插入文本。
<cite> 引述的作品标题，书 / 论文 / 散文 / 电影 / 歌曲 / 电视节目 / 画作。
<q> 引用的来自其他来源的段内内容，cite 属性表示该来源的 URL，比如名人名句，座右铭，伟人诗词等有来头的话语。
<abbr> 定义缩写，其 title 属性为全称。

4、微格式

<dfn> 用来展现一个术语的定义实例。
<time> 为表述的内容增加一个机器可读的时间数据，datetime 属性值必须是预定义的几种时间格式之一，如果不含 datetime 属性，则会解析其文本内容值作为时间数据。
<code> 代码片段。<kbd> 用户输入的内容 / 按键。<samp> 程序输出。
<span> 本身无语义，用来展现文本样式的最低优先级选择。
<mark> 在引用的文字中使用，表示在当前文档中需要引起注意但原文中并没有强调的含义，或表示与用户当前的行为相关的内容。比如对一篇文章的分析中对原文的标注或者按照关键字搜索出的文章中对该关键字的强调。

<ruby> 注音标示。
<rp> 标签在 ruby 注释中使用，以定义不支持 ruby 元素的浏览器所显示的内容。
<rt> 标签定义字符（中文注音或字符）的解释或发音。
主要用于东亚国家的文字注音。中日韩等。

用 HTML 已有的元素 / 属性，配合对属性值语义的扩展 (主要针对 class 属性) 以及对文档结构的约定来增强 HTML 的语义表达能力。Microformats 的规范本质上就是对一系列常用类型数据 HTML 格式的约定。

<hCard> 用来在 Web 上发布人物 / 公司 / 组织机构信息的格式。


<hCalendar> 发布事件
<hReview> 发布书评 / 影评 / 产品评价

5、语义化建议

  • （1）、尽可能少的使用没有语义的div和span元素。
  • （2）、在对语义要求不明显时，既能使用div也能使用p,那就使用p，因为p默认有上下边距，可以兼容特殊终端。
  • （3）、不要使用无语义的纯样式标签，如：b、font、u等，改用css设置。
  • （4）、需要强调的文本，可以包含在strong或者em标签中（浏览器预设样式，能用CSS指定就不用他们），strong默认样式是加粗（不要用b，因为没语义），em是斜体（不用i同b）；
  • （5）、使用表格时，标题要用caption，表头用thead，主体部分用tbody包围，尾部用tfoot包围。表头和一般单元格要区分开，表头标题用th，内容单元格用td；
